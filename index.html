Dynamic Proxy

动态代理指通过调用Proxy类的

来动态的创建代理的方法, 通过该方法创建的代理实现了interfaces中的接口, 当通过返回的代理调用方法时, 会转发调用至h: InvocationHandler上, 通过h来实现最终的调用

对interfaces的一些限制:

  • 其中的元素必须是接口
  • 不能有重复的元素
  • 所有的接口和内部的public方法必须在传入的classloader下通过名字可见

如果interfaces中元素顺序调换会得到两个不同的proxy实例

Generics

  • 在kotlin里面单独写一个代表的范型是nullable的, 也就是<Any?>, 为了保证non-null, 我们需要写成<T: Any>

Compose! (1)

知之为知之, 不知为不知, 是知也

纸上得来终觉浅, 绝知此事要躬行

compose的alpha版出来也有一段时间了, 最近想写一个安卓项目练练手(老不写脑子生锈了都), 既然compose出了, 那自然就不再写那堆恶心的xml了! 学习compose走起

Intro

Compose的设计逻辑和原生安卓View那一套是完全不同的, 在原生中我们一般使用findViewById()来获取我们需要的View的对象, 每个View是会维护一个internal state的, 我们可以说是“面向View对象”的编程

但在compose中, view是relative-stateless的, 他通过observe State类的变量来达到更新UI的目的, 这一步在compose中叫做recompose, 当事件发生状态改变时会尽可能少的全部重新绘制相关组件, 这种思想其实是非常适合与ViewModel进行协作的. 我们可以在widget的监听事件中更新ViewModel中的livedata, UI本身使用拓展函数LiveData::observeAsState得到的State来绘制自己, 这样在event发生后相关的widget会recompose

如上图所示, 这就是unidirectional pattern

Example

我们考虑一个简单的仅包含一个在button上显示其被点击次数的activity

我们考虑一个使用ViewModel的情况(在这个情形下是不必要使用viewModel的, 因为对数据的更新非阻塞直接在主线程进行, 传统方式直接在MainActivity里面维护一个count:Int就可以, compose方式在compose fun里面自己实现一个MutableState代理count变量就可以)

在传统方式中,

我们必须持有要更新的UI widget的对象, 并显式更新他们

在compose中,

可以看到我们的数据和UI是完全分离的, 我们把数据包装成了compose-observable的State对象, 在liveData中数据更新时, 所有observe它的compose fun都会自动的recompose, 我们无须再去显式的更新所有依赖于此数据的widget.

Scope fun

Context object
  • run, with, apply的context refer为this, 因为其参数为T.() -> R 形式, 所以能直接用this做引用
  • let, also 的context refer为it, 其参数为(T) -> R, 只不过在函数内部调用了block(this)
Return value
  • apply and also return the context object.
  • let, run, and with return the lambda result.
Facts

  • T.run{ } 和 with(T){ }等价

  • let相比于run和with区别在于它的参数block是(T)->R中可以执行T外部语句

  • apply和also区别也在于also的参数block可以执行T外部语句

  • 这个是一个experimental的功能, 用于告知编译器block只会被执行一次, 如果不加这个contract, block内部不可以对外部的val变量赋值

State Hoisting

caller持有callee需要的状态, 并且提供给callee其需要的更新状态的接口

state hoisting这一设计模式让我们可以写出纯stateless的composable, 并且把他们的状态hoist到上层

remember(){}会在compose内部保存状态, remember会在()内填入的参数不变的情况下不在recompose的时候重新执行{}这一block

72.Edit Distance

Posted on 2020-11-07 In leetcode

72.Edit Distance

source link

Description

Given two strings word1 and word2, return the minimum number of operations required to convert word1 to word2.

You have the following three operations permitted on a word:

  • Insert a character
  • Delete a character
  • Replace a character
Signature
Analysis

dp[i][j] represents minDistance(word1[:i], word2[:j])(inclusive)

if word1[i]==word2[j] => dp[i][j] = dp[i-1][j-1]

else dp[i][j] = min(dp[i-1][j]+1, dp[i][j-1]+1, dp[i-1][j-1]+1)

核心思想还是创建一个二维的dp数组, 只要找到递推表达式就简单了, 其中word1[i]!=word2[j]对应的三种情况就是:

  • dp[i-1][j]+1: word1往前退一个, 然后把word1[i]删掉(所以加1)
  • dp[i][j-1]+1: word2往前退一个, 然后把word2[j]插入进来
  • dp[i-1][j-1]+1: word1, word2都往前退一个, 然后把word[i]替换为word[j]
Solution

32.Longest Valid Parentheses

Posted on 2020-11-04 In leetcode

32.Longest Valid Parentheses

source link

Description

Given a string containing just the characters '(' and ')', find the length of the longest valid (well-formed) parentheses substring.

Signature
Analysis
  1. 在对数组进行操作的时候要时刻注意越界问题, 特别在循环中必须保证在access数组时, 在界内
  2. 在创建栈的时候一开始脑子抽了感觉能把长度设置为n/2+1省点空间… 被“(((((()”这个testcase教做人

接下来就是main idea了:

  1. 首先我们创建一个与字符串等长且初始为0的数组, 这个数组第i个元素只有在字符串这一位是’)’时可能非0, 代表了从这个’)’往前看最长连续的有效括号长度
  2. 在遍历的时候, 每当遇到’(’, 我们就把他的index压入栈中
  3. 如果遇到了’)’, 这意味着要consume一个’(’, 我们检查栈中是否为空, 如果是空的话就什么都不做(毕竟初始已经把长度设置为0了)
  4. 如果此时栈非空就consume一个’(’, 检查他前一位是否非0, 如果是的话就加上’(’前一位在array中的值
  5. 再加上这个’)’前一位在array中的值, 这对应了嵌套的情况, 即连续出现多个’)’
  6. 上面的步骤很大程度上都依赖了array初始值都为0, 这样就可以不用保留其他信息来随便加(如果是0的话无影响)
  7. 最后把array中最大元素返回就可了

感觉dp就是要充分利用之前的信息, 其实好像所有算法的目的都是要“customized”, 为特定的问题使用尽可能少的信息, 排除尽可能多的冗余信息来给出解

Solution

295.Find Median from Data Stream

Posted on 2020-11-03 In leetcode

295.Find Median from Data Stream

source link

Description

Median is the middle value in an ordered integer list. If the size of the list is even, there is no middle value. So the median is the mean of the two middle value.

For example,

Design a data structure that supports the following two operations:

  • void addNum(int num) - Add a integer number from the data stream to the data structure.
  • double findMedian() - Return the median of all elements so far.

Follow up:

  1. If all integer numbers from the stream are between 0 and 100, how would you optimize it?
  2. If 99% of all integer numbers from the stream are between 0 and 100, how would you optimize it?
Analysis

My approach

写了一个插入排序的, addNum是O(logn)+O(n)=O(n), findMedian是O(1), 但是这时空占用就离谱, 赶紧看大佬的解法

至于follow up我的想法是, 如果全部都是0-100之间那直接用一个100长的数组就好, add和find操作都是O(1)的; 如果是99%在0-100之间, 那么可以建一个100长数组然后比0小比100大部分可以直接用两个无序list保存

better apporach

看了用时较少的解法, 是用了两个堆来对半保存数组, 前半部分用大堆保存, 后半部分用小堆保存, 始终保持两个堆是balance的(类似于AVL中的概念, 此时是元素数目相差最多为1), 这样可以保持add复杂度为logn而find仍为O(1)

上面是我了解思路之后的具体实现

binary search

今天又被二分搜索搞懵了, 一个在random-access iterator上跑的二分搜索, 如果要保存一个该元素插入位置的话, 终止条件要设置为low<high, 要的就是low的范围可以到vector.size()上, 这样的话mid的求的条件应该为(low + high - 1) / 2, 使得mid取值始终不能等于high(以防止可能的越界问题), 并且设置low的时候要设置为mid+1, 这样一定不会陷入死循环.

这样结束之后low的值就是查找元素在数组中应占有的位置
